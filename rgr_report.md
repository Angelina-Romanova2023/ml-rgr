# Расчетно-графическая работа по дисциплине «Машинное обучение и большие данные»

## Тема
Разработка Web-приложения (дашборда) для инференса (вывода) моделей ML и анализа данных.

## Введение
Данная расчетно-графическая работа посвящена разработке интерактивного веб-приложения с использованием фреймворка Streamlit. Целью работы является демонстрация процесса анализа данных, обучения моделей машинного обучения и их использования для предсказания цен на алмазы. Проект включает в себя предобработку данных, обучение шести различных моделей регрессии, а также визуализацию данных и результатов предсказаний в удобном пользовательском интерфейсе.

## 1. Анализ данных и предобработка

Для выполнения работы был предоставлен набор данных `diamonds_processed.csv`, содержащий информацию о различных характеристиках алмазов и их ценах. Целевой переменной является `price` (цена алмаза).

### 1.1. Обзор данных

Набор данных содержит информацию о 20,000 записях (ограничено для ускорения обучения) и 10 признаках. Основные статистические характеристики данных представлены в таблице ниже.

| Feature | Type | Missing Values | Unique Values |
|---|---|---|---|
| carat | float64 | 0 | 252 |
| cut | int64 | 0 | 5 |
| color | int64 | 0 | 7 |
| clarity | int64 | 0 | 8 |
| depth | float64 | 0 | 184 |
| table | float64 | 0 | 127 |
| price | float64 | 0 | 11568 |
| x | float64 | 0 | 295 |
| y | float64 | 0 | 294 |
| z | float64 | 0 | 209 |

### 1.2. Предобработка данных

В процессе предобработки данных были выполнены следующие шаги:
*   **Масштабирование признаков:** Числовые признаки были масштабированы с использованием `StandardScaler` для приведения их к единому диапазону, что важно для корректной работы многих алгоритмов машинного обучения.
*   **Разделение на обучающую и тестовую выборки:** Данные были разделены на обучающую (80%) и тестовую (20%) выборки для оценки производительности моделей.

## 2. Обучение моделей машинного обучения

Для решения задачи регрессии (предсказания цены алмазов) были обучены шесть различных моделей машинного обучения. Выбор моделей соответствует требованиям задания и охватывает различные подходы: от классической линейной регрессии до глубоких нейронных сетей и ансамблевых методов.

### 2.1. Используемые модели

1.  **ML1: Линейная регрессия (Linear Regression)**: Простая и интерпретируемая модель, устанавливающая линейную зависимость между признаками и целевой переменной.
2.  **ML2: Градиентный бустинг (GradientBoostingRegressor)**: Мощный ансамблевый метод, строящий предсказания последовательно, корректируя ошибки предыдущих моделей.
3.  **ML3: Гистограммный градиентный бустинг (HistGradientBoostingRegressor)**: Усовершенствованная версия градиентного бустинга, оптимизированная для больших наборов данных.
4.  **ML4: Случайный лес (RandomForestRegressor)**: Ансамблевый метод, использующий множество деревьев решений и усредняющий их предсказания для повышения точности и устойчивости.
5.  **ML5: Стэкинг (StackingRegressor)**: Мета-ансамблевый метод, который использует предсказания нескольких базовых моделей в качестве входных данных для финальной модели (мета-модели).
6.  **ML6: Многослойный перцептрон (MLPRegressor)**: Полносвязная нейронная сеть, способная улавливать сложные нелинейные зависимости в данных.

### 2.2. Результаты обучения

Все модели были обучены на масштабированных данных. В качестве метрики качества использовался коэффициент детерминации $R^2$ (R-squared), который показывает долю дисперсии целевой переменной, объясняемую моделью. Чем ближе $R^2$ к 1, тем лучше модель.

| Модель | Название | $R^2$ Score |
|---|---|---|
| ML1 | Linear Regression | 0.7498 |
| ML2 | Gradient Boosting | 0.9305 |
| ML3 | HistGradientBoosting | 0.9547 |
| ML4 | Random Forest | 0.9526 |
| ML5 | Stacking Ensemble | 0.9477 |
| ML6 | MLP Neural Network | 0.9281 |

Как видно из таблицы, наилучшие результаты показали ансамблевые модели, в частности `HistGradientBoostingRegressor` и `RandomForestRegressor`, что ожидаемо для сложных задач регрессии.

## 3. Веб-приложение Streamlit

Разработано интерактивное веб-приложение с использованием фреймворка Streamlit, доступное по ссылке: https://8501-iceyoyfm1bc17c0cstuv4-b804a95f.us1.manus.computer. Приложение состоит из четырех основных страниц:

### 3.1. Developer Info (Информация о разработчике)

Эта страница содержит общую информацию о проекте и разработчике. Включает ФИО, номер группы, название университета, тему РГР и краткое описание целей проекта. Также перечислены основные технологии, использованные в разработке.

![Developer Info Screenshot](/home/ubuntu/developer_info.png)

### 3.2. Dataset Info (Информация о наборе данных)

На этой странице представлен подробный обзор используемого набора данных. Включает общее количество записей и признаков, статистическое описание данных (`df.describe()`), информацию о типах признаков, пропущенных значениях и количестве уникальных значений. Также отображаются ключевые метрики целевой переменной `price` (минимальная, средняя, максимальная цена) и краткое описание шагов предобработки.

![Dataset Info Screenshot](/home/ubuntu/dataset_info.png)

### 3.3. Data Visualization (Визуализация данных)

Страница визуализации данных демонстрирует ключевые распределения и зависимости в наборе данных с помощью четырех различных графиков:

1.  **Распределение цен на алмазы (Price Distribution)**: Гистограмма, показывающая частоту различных ценовых диапазонов.
2.  **Цена против первого признака (Price vs First Feature)**: Диаграмма рассеяния, иллюстрирующая зависимость цены от одного из признаков (например, `carat`).
3.  **Box Plot цен (Price Box Plot)**: Ящичковая диаграмма, отображающая медиану, квартили и выбросы в распределении цен.
4.  **Корреляционная матрица признаков (Feature Correlation)**: Тепловая карта, показывающая взаимосвязи между всеми числовыми признаками в наборе данных.

![Data Visualization Screenshot](/home/ubuntu/data_visualization.png)

### 3.4. Model Prediction (Предсказание моделей)

На этой странице пользователи могут взаимодействовать с обученными моделями для получения предсказаний. Представлены R²-метрики для каждой из шести моделей. Пользователь может выбрать один из двух методов ввода данных:

*   **Ручной ввод (Manual Input)**: Ввод значений для каждого признака через интерактивные поля. После ввода модель делает предсказание цены.
*   **Загрузка CSV (Upload CSV)**: Загрузка CSV-файла с данными для предсказания. Приложение обрабатывает файл, делает предсказания для каждой строки и позволяет скачать результаты в новом CSV-файле.
